### 特に高パフォーマンスの上位3投稿

```
🥇 年齢のせいじゃなかった   → 402ビュー / ER 0.75%
🥈 婚活やめたいと思った日   → 335ビュー / ER 0.60%
🥉 夜中に泣いた話          →  93ビュー / ER 1.08%
```

### なぜtrust_buildingが効くのか｜3つの構造的理由

**① 「検索意図×感情一致」の法則**
> 「年齢のせいじゃなかった」「婚活やめたい」は、婚活ユーザーが**心の中で検索している言葉**そのもの。Threadsのアルゴリズムは滞在時間を重視するため、タイトルで感情を捕まえた投稿が拡散されやすい。

**② ミキペルソナの「6年・100人」という数字の権威性**
> 一般的な婚活アドバイスは「専門家が言う正論」だが、ミキの投稿は「同じ地獄を経験した人の証言」。読者にとって**「私のことを知っている人だ」**という錯覚を生む共感型コンテンツが機能している。

**③ ネガティブタイトルの高CTR効果**
> 「泣いた」「やめたい」「消耗した」などの感情的ネガワードは、スクロールを止める力が強い。Threadsでは冒頭2行が折りたたまれるため、**タイトル＝最初の2行の引力**が全てを決める。

## 3. 改善が必要な投稿タイプと具体的改善案

### ❌ advice型：ER 0.0%・ビュー15という壊滅的結果

#### 根本的問題の診断

```
問題の本質：
「アドバイスをくれる人」より
「私と同じ経験をした人」として
信頼を獲得する段階にある
```

**現時点でadvice投稿が機能しない理由：**
- フォロワー数が少ない＝まだ「専門家として認知」されていない
- タイトル不明（空白）＝アルゴリズムに拾われない
- 「正しいこと」より「共感できること」がThreadsで伸びる

#### 具体的改善案

**短期（今すぐ）：adviceはtrust_buildingに包んで出す**

```
❌ Before（純粋advice）
「マッチングアプリで失敗する人の3つの特徴」

✅ After（trust_building型adviceに転換）
「100人以上と会って全滅した私が、
 唯一変えたこと」
→ 体験談として入り→自然にadviceを埋め込む
```

**中期（フォロワー500人超えてから）：**
advice型を独立投稿として復活させる。その際も必ず「私自身の体験」をアンカーとして冒頭に置く。
