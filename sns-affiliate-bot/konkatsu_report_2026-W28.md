## 3. 改善が必要なタイプと具体的改善案

### ❌ Problem 1：`advice`タイプが壊滅的（15views・ER0%）

**原因分析：**
- タイトルが空白（データ上`"title": ""`）→ Threadsでのインプレッション獲得機会をゼロにしている
- 「アドバイス」は初期アカウントには早い。**信頼残高がない段階での説教は離脱を生む**

**改善案：**
```
❌ やってはいけないこと
→ 「婚活でやるべき3つのこと」「プロフの書き方」等の一般論

✅ やるべきこと
→ adviceをtrust_buildingに「溶かす」
例：「アドバイスを言いたいわけじゃないんだけど、
    私が8ヶ月で変わった理由を正直に話す」
    という入口からadvice要素を後半に忍ばせる構成
```

### ❌ Problem 2：ER0%投稿5本の共通点

```
・自分を偽って疲れた（13views）
・6年間で失ったもの（15views）
・プロフ修正しても変わらなかった（9views）
・メッセージで消耗した2年間（5views）
```

**パターン分析：**

| 問題点 | 詳細 |
|--------|------|
| タイトルが「過去の消耗」止まり | 読み手に「で、どうなったの？」という先への期待値がゼロ |
| ビュー数が一桁〜十数件 | 投稿直後の初速が出ていない＝投稿時間・ハッシュタグ設計の問題 |
| 感情テーマが重複している | 「疲れた」「消耗」「失った」が近い感情域に集中しすぎて差別化できていない |

**具体的改善策：**

```markdown
【タイトル改善の公式】
Before：「自分を偽って疲れた」
After：「自分を偽い続けた私が、やめた瞬間に起きたこと」
         ↑疲れた（過去）        ↑転換点（未来への期待）

Before：「6年間で失ったもの」  
After：「6年で失ったものより、8ヶ月で得たものの話」
         ↑損失（重い）              ↑対比（希望）

Before：「メッセージで消耗した2年間」
After：「2年間毎日メッセージして0人と会えた私が気づいた盲点」
         ↑数字で具体化                ↑「盲点」で知的好奇心
```

### ❌ Problem 3：リプライが全投稿で0件

**最重要課題。** Threadsのアルゴリズムはリプライを最重視する。

**改善策：投稿末尾に「返答しやすい問い」を必ず設置**
```
例文：
「婚活中に一番しんどかった瞬間って、どんな時でしたか？
 よかったら教えてください🌿」

「年齢が気になって動けなかったこと、ありますか？」
```
