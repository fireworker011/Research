# 3ヶ月ロードマップ — 40代パパの「表と裏」二層構造で月利30万を狙う

**作成日**: 2026-07-12 ／ **対象期間**: 2026-07-13（月）〜 2026-10-12（Day 1〜90）

## この計画の結論（先に全部言う）

1. **「裏の自動化プロジェクト」= このリポジトリの affiliate-engine は既にほぼ無人運転が完成している。**
   人間の作業は1日10分のコピペだけ。つまり「リソース30%を裏に割く」という制約は、
   **設計上すでにクリアされている**。30%枠の実態は「週末の判断作業60分＋日次10分」で収まる。
2. **表（X＋YouTube）は、裏の実測数値を発信ネタにする。** 「AIと自動化で裏に収益エンジンを
   作っている40代パパ」というストーリーは作り話ではなく、このリポジトリの稼働ログそのものが証拠になる。
   毎日自動生成されるKPIレポートが、そのまま毎日の発信ネタになる（＝ネタ切れが構造的に起きない）。
3. **ペットカメラYouTubeは「収益源」ではなく「信頼性の担保」として使う。**
   登録50人でも平均再生2,000回/本×18本継続は、「顔出しなしで回るチャンネルを実際に持っている」
   という反証不能なプレイヤー実績。詳細は [03_pet-camera-bridge.md](03_pet-camera-bridge.md)。
4. **正直な着地予測**: 3ヶ月で月利30万は上振れシナリオ（README の前提と同じ）。
   中央値は月5〜15万。ただし本計画は「30万に届かなかった場合に何が足りないかを数字で特定できる」
   設計になっており、その検証過程自体が表の発信コンテンツになる（＝どちらに転んでも資産が残る）。

## 絶対に守る3つのルール（先に決めておく）

1. **数字を盛らない。** プロセスエコノミーは「小さい実数を隠さず見せて過程で信頼を得る」モデル。
   登録50人を恥じる必要はなく、むしろ「50人からどう伸ばすか」が商品になる。
   誇張は一度バレたら全資産（発信の信頼）が消える。
2. **表と裏を紐づけない。** Threads の9アカウント名・ハンドルは表の発信で絶対に出さない
   （同一運営者と機械的に紐づくと全アカウントのリーチ・生存に関わる。account-strategy.md の方針と同じ）。
   表に出すのは「数値・アーキテクチャ・学び」だけ。
3. **法規制はシステムに任せ、表の発信でも同じ基準を守る。** 「必ず稼げる」等の断定は
   compliance.js が裏でブロックしているのと同様、表のXでも書かない（景表法）。
   教材販売時は特商法表記を用意する。
