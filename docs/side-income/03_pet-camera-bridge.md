# ペットカメラYouTubeの実績を「40代パパのAI自動化ストーリー」に組み込む導線設計

## 発想の転換: 「登録50人」は弱みではなく、切り口を変えれば最強の実績になる

同じ事実でも、どの数字を前に出すかで意味が変わる。**すべて実数のまま**、以下のように語る:

| 事実 | 弱い見せ方（やらない） | 強い見せ方（やる） |
|---|---|---|
| 登録50人・平均再生2,000回 | 「登録者50人です」 | 「**登録者の40倍が毎回見に来るチャンネル**を運用している（平均再生2,000回/本）」 |
| 18本投稿・継続中 | 「18本投稿しました」 | 「会社員をしながら**18本連続で継続**。仕組み化したから続いている」 |
| 顔出しなし・ペットカメラ素材 | 「ペットの動画です」 | 「**顔出しゼロ・撮影ゼロ**（カメラが勝手に撮る）で回る、再現性のある型」 |
| 婚活YT 3本・登録3人 | （隠す） | 「**失敗した方も公開する**: 同じやり方で当たらなかったチャンネルとの差分がこれ」 |

ポイントは3つ:
1. **平均再生2,000回は登録50人に対して異常値**であり、「登録者数ではなく検索・ブラウジング流入で回る設計」の証拠。これは「フォロワーゼロから始める人」にとって最も知りたい情報＝コンテンツ価値がある
2. **「継続できている」こと自体が、失敗恐怖症の40代がターゲットのアカウントでは最大の実績**
3. 婚活YTの失敗も出す。成功だけの発信者より「当たり外れの差分を実測で語れる」発信者の方が信頼される（プロセスエコノミーの本質）

## 導線の全体図

```
                     【信頼性の担保（実在の証拠）】
ペットカメラYT ──────────────────────────────┐
  概要欄・コミュニティ投稿 → X へ              │
                                              ↓
X（表の本丸）: 固定ポスト「実績ダッシュボード」 ← 毎週日曜更新
  ├ ドキュメント型ポスト（Day N の実測数字）
  ├ 学び型ポスト（裏エンジンから得た知見）
  └ プロフィールリンク → lit.link 等
        ├ ペットカメラYT（誰でも確認できる実績）
        ├ note 無料記事（教育）
        └ メルマガ/LINE（リスト・Month 2〜）

【裏】Threads 9垢・affiliate-engine → 数値・アーキテクチャ・学びだけをXに供給
     （アカウント名・ハンドルは絶対に出さない）
```

## X 固定ポスト: 「実績ダッシュボード」の型

固定ポストは自己紹介文ではなく**毎週更新される数字の一覧**にする。これが「プレイヤー感」の正体。

```
40代・子持ち会社員。通勤電車のスマホだけで
AI自動化の収益エンジンを作る過程を全部公開しています。

📊 今週の実測（毎週日曜更新）
├ YouTube: 18本 / 平均再生2,000回（登録50人の40倍）
├ SNS自動投稿エンジン: 9アカウント・4回/日 無人稼働
├ 直近views: 13,609/週 → 目標 62.5万/月（進捗2.2%）
├ 収益: ¥0（8/5 リンク解禁予定）
└ 作業時間: 平日45分（通勤中のみ）＋週末3h

失敗も全部出します。Day 1 は 2026-07-13。
```

- **進捗2.2%のような「遠さ」も隠さない**。ゴールとの差分を毎週見せるから、埋まっていく過程に価値が出る
- 数字は `metrics_history.jsonl` と日次レポートから転記するだけ（週次ポスト文面の自動生成は → 04）

## ペットカメラYTを実績として使う具体ポスト例（すべて実数ベース）

1. **アーキテクチャ公開型**: 「登録50人なのに平均再生2,000回のチャンネルの中身: 撮影0分（ペットカメラが撮る）→ 切り出し→ サムネはStable Diffusionの型を使い回し → 週末60分で1本。仕組み図解↓」
2. **差分分析型**: 「同じ手法で2チャンネル作って、片方は平均2,000再生、片方は3本で止まった。差は◯◯だった」（婚活YTの失敗を活かす）
3. **リアルタイム型**: 「今週の1本、通勤中にスマホでタイトルだけ3案考えた。どれがいいと思う？」（制作過程を見せながら質問で締める）

## YouTube側の導線設置（Week 3 に一度だけやる作業）

- 全動画の概要欄テンプレに1行追加: 「このチャンネルの運用の裏側（仕組み・数字）はXで公開中 → @xxx」
- コミュニティ投稿（登録50人でも使える場合のみ）で月1回、Xの実績ダッシュボードのスクショを共有
- **動画の中身は変えない**。視聴者はペットを見に来ている。導線は概要欄に置くだけで、本編に副業の話を混ぜない（チャンネルの数字を守ることが実績の維持そのもの）

## Threads ペットアカウントとの連携（裏側のシナジー）

`config/accounts.json` に設定済みの pet アカウント（裏）は、ペットカメラYTの素材・ネタと相性が良い。
ただし**表のXとは接続しない**。裏のpetアカ → YT誘導は可（Threadsの投稿にYTリンクを混ぜる形で、
リンク解禁後に `links.json` の運用に乗せる）。方向は常に「裏→表の資産へ流す」の一方向のみで、
逆方向（表→裏アカの紹介）は絶対にやらない。

## やってはいけないことの再確認

- 「月30万稼いでいます」等、**未達の数字を達成済みのように書くこと**（景表法リスク＋信頼の全損）
- 登録者数・再生数の水増し表現（「約」でごまかさず実数を書く）
- ペットカメラYT本編への副業コンテンツ混入（既存視聴者の離脱＝実績の毀損）
- 他人の実績・事例を自分の実績のように引用すること
