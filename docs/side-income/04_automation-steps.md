# 省力化・自動化ステップ — 集客・教育・ローンチを30%バッファを圧迫せずに回す

方針: **裏で既に動いている affiliate-engine の仕組みを、表（X・note・ローンチ）に横展開する**。
新しいツールの契約・学習コストは増やさない（GitHub Actions + Claude API + 既存スクリプトの拡張のみ）。

## 現在すでに自動化されているもの（追加作業ゼロ）

| 領域 | 仕組み | 人間の作業 |
|---|---|---|
| Threads 投稿 | affiliate_engine_post.yml（4回/日・分散付き） | 0分 |
| 改善ループ | affiliate_engine_insight.yml（毎朝6時: 実測×Web検索→テンプレ自動更新） | 0分 |
| KPI計測 | affiliate_engine_report.yml（毎日14時: レポート+着地予測） | 眺めるだけ |
| エンゲージ下書き | engage.js → GitHub Issue に毎朝配達 | コピペ送信10分 |
| トークン管理 | refresh_threads_token.yml（週次自動リフレッシュ） | 0分 |

## 追加する自動化（優先順・すべてこのリポジトリ内で完結）

### Step 1（Week 2〜3）: X下書きの自動生成 — 毎朝のIssueに「X用3本」を追加する

**効果が最大。** 表の運用から「書く」作業が消え、平日のXタスクが「選んで数字を差し込んで投稿」の5分になる。

- 実装: `engage.js` と同じ方式で新スクリプト `x-drafts.js` を追加し、insight ワークフローの末尾に組み込む
- 入力: 当日の `metrics_history.jsonl`（実測数字）＋ 直近レポート ＋ 発信軸のプロンプト（40代パパ・プロセスエコノミー・盛らない）
- 出力: 毎朝のIssueに以下の3型で下書き3本
  - **数字報告型**: 「Day N。今週のviews ◯◯（先週比+◯%）。やったことは〜」
  - **学び型**: 裏エンジンの改善アクション（insight レポートの知見）を一般化して共有
  - **質問型**: フォロワーとの会話誘発（Threadsで実証済みの「質問締め」をXにも適用）
- ガード: Threadsアカウント名・ハンドル・ジャンル構成が特定できる固有情報は出力に含めない（プロンプトで禁止＋出力チェック）。compliance.js のNG表現チェックを通してから出力する
- **X APIによる自動投稿はしない**（有料枠・凍結リスク・「投稿ボタンは人間」の原則は表でも同じ）

### Step 2（Week 4）: 週次ダッシュボード文面の自動生成

- 実装: `report.js` に週次モード（日曜実行）を追加。1週間の実測を集計し、
  固定ポスト用の「実績ダッシュボード」文面（→ 03 の型）をそのまま出力
- 人間の作業: 日曜に文面をコピペしてX固定ポストを差し替える（5分）

### Step 3（Week 4〜Month 2）: note 記事の自動下書き

- 実装: 月2回、insight レポート2週間分＋metrics を入力に、note記事構成（見出し＋本文8割）を生成して
  `output/note_drafts/` に保存（Issue にも通知）
- 人間の作業: 週末に30分で推敲・スクショ差し込み・公開。**ゼロから書かない**
- ネタは常に実録: 「初クリックまでの14日間の実測」「ワーストジャンルを止めた判断の裏側」など、
  裏エンジンのログがそのまま記事になる

### Step 4（Month 2）: リスト取りの省力設計（教育の自動化）

- ツールは無料枠で始める: LINE公式（無料枠200通/月） or メール配信の無料プラン。**Lステップ等の有料高機能ツールはDay 90まで契約しない**
- あいさつメッセージ＋ステップ配信3通（自己紹介→実測データの深掘り→教材予告）を一度だけ設定
  → 以後、登録者への教育は全自動
- リードマグネット: 「実測データの生スプレッドシート」or「自動化アーキテクチャ図解PDF」
  （既にあるものを整形するだけ。新規制作しない）

### Step 5（Month 3）: ローンチの省力設計

- 商品: note有料記事 or Brain。**決済・配信・返金対応をプラットフォームに全部任せる**（自前決済・自前会員サイトはやらない）
- 教材の中身は新規執筆しない: ここまでの note 無料記事＋実測データ＋リポジトリのアーキテクチャ解説を
  再構成し、「実録＋再現手順」としてパッケージ化（推敲は週末2回分＝3時間で完成させる）
- プレローンチのX投稿もStep 1の仕組みで下書き生成（「教材スライス公開型」をテンプレに追加）
- 特商法表記・「成果を保証しない」旨の明記を必ず入れる（compliance.js と同じ基準を人間側でも守る）

## 自動化しないと決めるもの（30%バッファを守るための境界線）

| やらないこと | 理由 |
|---|---|
| X の自動投稿・自動いいね・自動フォロー | 凍結リスク＋API費用。「下書きは自動・送信は人間」の上限を表でも守る |
| YouTube の投稿自動化・量産拡大 | 週1本×60分が品質と継続の均衡点。拡大はDay 90以降に判断 |
| DM でのセールス（手動含む） | 時間対効果が最悪で30%バッファを最も圧迫する。導線はプロフィールリンクに一本化 |
| 高機能マーケツール（Lステップ有料版等）の導入 | 月額費用が先行し、設定作業が週末を食い潰す。リスト500人まで不要 |
| 表用の新規自動化基盤の構築 | 既存エンジンの拡張で全部賄える。二重運用は保守時間を倍にする |

## 実装ロードマップまとめ（裏の30%枠でやる開発作業）

| 時期 | 実装 | 見積もり |
|---|---|---|
| Week 2-3 | Step 1: x-drafts.js ＋ insight ワークフロー組み込み | 週末1回 |
| Week 4 | Step 2: 週次ダッシュボード出力 | 週末0.5回 |
| Week 4〜 | Step 3: note 下書き生成 | 週末0.5回 |
| Month 2 | Step 4: ステップ配信の初期設定（開発ではなく設定作業） | 週末1回 |
| Month 3 | Step 5: 教材パッケージ化（人間の推敲がメイン） | 週末2回 |

→ 開発は Month 1 に集中し、Month 2 以降の裏枠は「週次判断20分＋保守」に戻る。
これにより Month 2〜3 の30%バッファは本来の「別件の裏作業」にフルに使える。
