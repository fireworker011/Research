# YouTube Shorts 第三弾 — 戦略とジャンル選定（2026-07 リサーチ版）

第1弾（ペット）・第2弾（婚活）に続く第三弾チャンネルの立ち上げ方針。
市場リサーチ〜投稿ネタ出しは `src/shorts-research.js` が毎朝自動で行い、
採用・動画化・アップロードの判断だけを人間が行う（半自動）。

## 運用フロー（半自動の全体像）

```
毎朝 自動（GitHub Actions affiliate_engine_shorts.yml）
  shorts-research.js
    ├ チャンネル別に Web検索で市場リサーチ（伸びている傾向・フック・型）
    ├ 同ジャンル Threads アカウントの実測を補助シグナルとして突き合わせ
    ├ 台本レベルのネタを生成（タイトル/フック/ナレーション/タグ/CTA/リンク）
    ├ compliance.js で誇大表現・薬機法リスクをブロック
    └ Issue「🎬 YouTube Shorts ネタ出しキット」に配信 ＋ 採用待ちJSONを保存
        ↓
人間（10〜30分・週2〜3回でも可）
  キットからネタを採用 → node src/video-semi-auto.js --ideas で確認・動画化
        ↓
  YouTube Studio で予約投稿（概要欄に link_key のアフィリエイトリンク+#PR）
```

## 第三弾ジャンルの推薦: 節約・お金

事前リサーチ（2026-07 時点の公開情報）に基づく推薦。採用前に
`node src/shorts-research.js --select-genre`（または Actions の手動実行で
`select_genre=true`）を回すと、Claude の Web 検索による最新の比較レポートが
`output/shorts/genre_research_*.md` に出る。

### 推薦理由

1. **顔出しなし Shorts と相性が良い**: 2026年の「稼げるジャンル」解説では、教育・学習系や節約・時短系の実務情報が顔出しなしで伸びやすいジャンルとして繰り返し挙がっている。お金の雑学系 Shorts（銀行手数料・制度の落とし穴など）で数十万再生の事例もある
2. **既存アセットが使える**: `links.json` に 節約_格安SIM / 節約_ふるさと納税 のリンクキーが定義済みで、Threads の節約アカウント（2026-07-08 開設）の実測データを補助シグナルとして流用できる
3. **既存2チャンネルと客層が重ならない**: ペット（癒し）・婚活（恋愛）に対しお金（Money）は HARM ジャンルの残る柱で、ポートフォリオとして分散が効く
4. **成果地点が浅い案件がある**: 格安SIM乗り換え・ふるさと納税は「申し込み」まで距離が短く、Shorts の軽い視聴からでも転換しやすい

### 対抗候補（比較）

| 候補 | 強み | 見送り理由 |
|---|---|---|
| 睡眠 | マットレス等の高単価案件 | 商材が高額で Shorts の衝動的視聴と距離がある。比較検討型で長尺向き |
| 教育（習い事） | Threads で会話が起きやすい実績 | 教室系は地域性が強く、Shorts の全国配信と成果地点のミスマッチ |
| 副業 | 案件数が多い | 誇大系チャンネルが飽和し規制リスクも最大。Threads 側で既に運用中 |

### チャンネル設計（たたき台）

- **コンセプト**: 「知らないと損する系」のお金の雑学×実践。1本1トピック、冒頭2秒で損失回避を提示（例:「その手数料、払わなくていいです」）
- **フォーマット**: テキスト+ナレーション+フリー素材/スクリーン録画。顔出しなし
- **名前の方向性**: 「◯◯｜1分で分かるお金の話」「知らないと損するお金ラボ」系
- **投稿ペース**: 開設2週間は1本/日でアルゴリズムに学習させ、以降は反応を見て調整
- **導線**: 価値提供3〜4 : リンク誘導1。誘導は格安SIM・ふるさと納税の「今日やれる手続き」文脈のみ
- **⚠️ 法規制**: 金融は誇大表現規制が最も厳しい。利益断定・元本保証系は compliance.js がブロックするが、投資の個別推奨は書かない（ネタ生成プロンプトでも禁止済み）

### 採用手順

1. `output/shorts/genre_research_*.md`（--select-genre の結果）を確認し、推薦を採用するか決める
2. 別ジャンルにする場合は `config/youtube.json` の第三弾チャンネルの `genre` / `threads_genre` を書き換える
3. チャンネル開設後、`handle` を記入し `status` を `candidate` → `active` に変更
4. 翌朝から他チャンネルと同列でネタ出しキットに載る（candidate の間も「開設検討中」印付きでネタは届く）

## KPI と撤退ライン

- Shorts の収益はアフィリエイト（概要欄リンク）が主。再生数は手段であって目的ではない
- 開設30日: 平均再生が2チャンネル既存実績の半分未満 → フック・型を全面見直し（ネタ出しキットの winning_formats を再確認）
- 開設60日: リンククリックがほぼゼロ → 案件かCTA文脈のミスマッチを疑い、link_key の差し替えを検討
