# 3ヶ月ロードマップ & 日々のタスク設計（リソース7割・実施率100%版）

対象: 40代・子持ち会社員 / 通勤スマホのみ / 過去の副業失敗で失敗恐怖症
作成日: 2026-07-12（改訂）

## この設計の前提（重要）

- **可処分時間の30%は「別件の裏作業」用の固定ブロックとして最初に確保する。** この文書が設計・最適化するのは残り**70%（表向きのXマネタイズ・YouTube・AI効率化発信）だけ**。裏側の中身には踏み込まない。30%は"埋めるもの"ではなく"守るもの"として扱う。
- したがって表の設計思想は一貫して**「7割の力で、10割の成果に見えるように回す」**＝徹底的な省力化・自動化・"やらないこと"の明確化。
- **月利30万は上振れシナリオ**（既存READMEの正直な明記と同じ）。リソース7割・スマホのみなら現実的着地は月1〜10万台。この3ヶ月のゴールは「30万を約束する」ことではなく、**"どの型・どの案件が自分に効くか"を実測で確定させ、勝ち型に集中できる状態を作る**こと。30万はその延長線上。

## 1. コンセプト（プロセスエコノミー）

軸: **「崖っぷち40代パパが、AIと自動化で"裏で仕組みを作っていく"過程そのものを見せる」**。
プロセスエコノミー＝結果が出る前から"作っている過程"を発信して信頼と実績を積む型。あなたには既に**ペットカメラという"実際に手を動かして反応が出ている証拠"**があるので、この型と相性が最強。

表で見せるのは合法・健全な範囲（AI効率化・ペットカメラ運用・副業の実測）だけ。裏の本丸は一切出さない。**「AIで色々自動化して実験している人」という人物像**が、そのまま高単価AI副業商材への信頼になる。

## 2. 3ヶ月ロードマップ（月次・リソース7割前提）

### Month 1 —「土台と収益化スイッチ」（伸ばすより、繋ぐ）

新規制作を最小化し、既にある資産を"収益が発生し得る状態"にするだけ。

- **W1**:
  - X本体アカウントを「崖っぷちパパ｜AIと自動化の記録」系に整備。プロフィールに「ペットカメラ運用中／AIで副業を実験中」。
  - A8.net で提携申請。`config/links.json` の **ペット_Furbo / ペット_保険 / ペット_フード** と、**副業/AI/教育系の高単価（1件1万円超）**を最優先で1つ含める。
  - `data/conversions.csv` を作成（空でよい。無いと `report.js` が収益を計測できない）。
- **W2**:
  - ペットカメラ18本の概要欄・固定コメントに短縮URL（クリック計測付き）でアフィリンク一括設置。**これが最速の1円の発生源**。
  - Threads は **education垢に集中**、他4垢は自動運転のまま放置（手をかけない）。
  - X で「ペットカメラ運用の裏側＝AI活用の実演」を週3ポスト（下書きは既存の `engage.js` 素材を流用）。
- **W3-4 / Day30 中間評価**:
  - 短縮URLの実測クリックを `report.js` 日次で確認。
  - **確定¥10,000未満なら → 案件（オファー）を入れ替える**（ジャンルは変えない。READMEの判断ルール）。

### Month 2 —「勝ち型の特定と横展開」

- ペットカメラで**再生が伸びた動画の"型"**（サムネ・冒頭・尺）を3本抽出 → 同じ型で量産（バズは再現）。
- X で伸びたポストを `video-semi-auto.js` で Shorts/Reels 化 → YouTube/X に横展開（Threads外流入）。
- **Day45-60 評価**: 月次確定¥30,000が現実的な良ライン。超えていれば型は正しい。

### Month 3 —「高単価1本への集約」

- 2ヶ月で最も成約が出たジャンル1つに絞り、高単価案件（AI副業/プログラミング教育スクール、1件1〜3万円級）を接続。
- 物語を**「0円→ここまで来た実録」**として使う。**実測の数字がそのままセールスコピーになる**（盛らないから刺さる）。
- 30万未満でも、**「どの型・案件・プラットフォームが自分に効くか」の地図**が完成。これが本当の資産。

## 3. 日々のタスク設計 —「7割の力・実施率100%」の3原則

失敗恐怖症×時間なし×スマホのみ×30%は裏で確保、の全条件を満たすには**「意思」に頼る設計を全部捨てる**のが唯一の答え。

### 原則1: まず30%ブロックを"守る"。表は残り70%に必ず収める
1日の可処分時間を仮に**平日90分**とすると、内訳は固定:
- **裏（本丸）: 約27分/日 = 死守。ここは絶対に削らない・侵食させない。**
- **表（この設計）: 約63分/日が上限。これを超えたら設計ミスと見なす。**

表の作業がこの上限を超えそうになったら、**タスクを増やすのではなく"やめる"**（後述の"やらないこと"）。パンクの99%は「表を頑張りすぎて裏を潰す」形で起きる。

### 原則2: 表の作業は「通勤電車の1駅=1タスク」に固定（意思に頼らない）
| タイミング | タスク | 所要 |
|---|---|---|
| 家を出て最初の1駅 | GitHub Issueのエンゲージ下書きをコピペ送信 | 3分 |
| 次の1駅 | 周辺投稿に いいね10〜20 / コメントした相手をフォロー | 3分 |
| 座れたら | ペットカメラ次動画のネタを1行メモ | 4分 |
| 帰りの電車 | X に1ポスト（朝AIが用意した下書き） | 3分 |

表の実作業は**平日13分前後**。残りの表リソースは動画編集など週末にまとめる。**"やる気のある日に頑張る"を禁止し、毎日同じ最小量に固定**するのが100%の作り方。

### 原則3: 週次判断は「日曜10分」だけ、数字に◯×を付ける
- `output/reports/` の最新日次 → 平均views/投稿が全体の半分未満のジャンルに×。
- 週1で ASP管理画面 → `data/conversions.csv` に成果を1行追記。
- **判断は感覚でなく常に数字で。** これが失敗恐怖症への薬（感情での撤退/深追いを止める）。

### やらないこと（＝70%を守るための削り）
- Threads 5垢を手動で伸ばそうとしない（自動運転のまま。型が出るまで母数は無意味）。
- 婚活の深追いをしない（162viewsは弱い。捨て枠）。
- 毎日の動画制作をしない（週末にまとめて。平日はネタメモだけ）。
- 新しいプラットフォームに手を広げない（X・YouTube・Threadsの3つで固定）。

## 4. ペットカメラの好調数値を「40代パパの副業・AI自動化ストーリー」に組み込む導線設計

あなたの資産で**唯一"実測で勝っている"のがペットカメラ**。これを"プレイヤー感（実際に手を動かして結果を出している人）"の証拠として物語の中心に据える。

```
ペットカメラ動画（登録50・平均2,000再生／家族向けの信頼できる入口）
   │  概要欄・固定コメント
   ├─▶ ①即時収益: Furbo/ペット保険/ペットフードのアフィリ（links.json 設置済みスロット）
   │
   │  「この運用、実はAIと自動化でほぼ手を離してます」の一言でXへ送客
   ▼
X「崖っぷちパパ｜AIと自動化の記録」（プロセスエコノミー本体）
   │  ペットカメラ運用の自動化＝AI活用の"実演"として発信
   │  （見せられる部分だけを見せる。数字が実在するから嘘がいらない）
   ▼
高単価アフィリ（AI副業スクール/プログラミング教育/ガジェット・1件1〜3万円級）
   「自分がやってる方法を体系的に学びたい人向け」の自然な誘導
```

**なぜ信頼性（プレイヤー感）に効くか:**
- ペットカメラは**"やります"ではなく"もう2,000再生出てます"という完了済みの実績**。プロセスエコノミーで最も足りなくなる「本当に手を動かしてる人なの?」への回答が最初から手元にある。
- 「AIでペットカメラ運用を自動化している」は**そのまま高単価AI副業商材の実演**になる（実演＝最強のセールス）。
- 家族向け・ほっこり文脈が40代パパ像と完全一致。物語に無理がない。

**具体アクション（今週・表リソース内で完結）:**
1. ペットカメラ概要欄テンプレ（固定文＋短縮URL）を作り、18本に一括設置。
2. Xプロフィール整備＋固定ポストにペットカメラ動画を貼る。
3. `config/links.json` の ペット_Furbo / 保険 / フード に実リンク設定（A8提携後）。

## 5. 30%バッファを圧迫しない「省力化・自動化」ステップ

表の情報発信・集客・教育・ローンチを、**極力あなたの手を経由させない**ための仕組み。既存システムに大半が実装済みなので"使うだけ"。

| 領域 | 省力化の仕組み | あなたの手作業 |
|---|---|---|
| 投稿（集客） | `threads-poster.js` が4回/日自動投稿（GitHub Actions cron） | 0分 |
| 改善（教育=型の進化） | `insight.js`→`apply-proposals.js` が毎朝自動でテンプレ改善・スケジュール再生成 | 0分 |
| 返信・コメント下書き | `engage.js` が毎朝GitHub Issueに下書き生成 | コピペ送信のみ |
| KPI計測 | `report.js` が毎日14時に日次レポート自動生成 | 日曜10分の確認 |
| 動画化（横展開） | `video-semi-auto.js` で反応上位を Shorts/Reels 化 | 週末まとめて |
| X下書き | 朝の自動生成素材を流用（新規ライティングをしない） | コピペ投稿 |

**ローンチ（高単価接続）の省力化**: Month3で商材を接続するとき、新しいセールス文を書かない。**2ヶ月間の実測ポスト・数字がそのまま"実績の積み上げ＝セールス"**になっているので、固定ポストとプロフィールリンクを差し替えるだけで"ローンチ"が完了する設計にする。

**鉄則**: 表で新しい手作業を思いついたら、まず「これは自動化済みの何かで代替できないか」を問う。できないなら"やらない"側に倒す。**表の手作業を増やすことは、そのまま30%バッファを削ること**だと常に意識する。

## 6. 撤退・転換ライン（失敗恐怖症のための安全装置）

| 時点 | ライン | アクション |
|---|---|---|
| Day30 | 確定¥10,000未満 | 案件（オファー）を総入れ替え。ジャンルは変えない |
| Day45 | ペットカメラ導線のCTR<0.3% | 概要欄CTA・サムネを全面見直し |
| Day60 | 確定¥30,000未満 | プラットフォーム or 商材の根本転換を検討 |
| 随時 | 表の作業が63分/日を超える | タスクを増やさず削る（30%バッファ死守） |
| 随時 | 特定アカのリーチ急落 | 数日投稿を減らして回復待ち（シャドウバン兆候） |

**「30万に届かない=失敗」ではない。** 3ヶ月で"自分に効く型の地図"が完成し、かつ30%バッファを一度も潰さずに走り切れたら、それは成功。
