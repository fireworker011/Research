# インサイトレポート 2026-07-22

## 筋トレ

**市場トレンド（Web検索による）**
- 単発の「バズ狙い」より、なぜそうなるかを分解し再現方法を示す“設計思考”の投稿が伸びやすい傾向。2026年は毎回、学びを1つ入れる・思考の背景を説明する・なぜそうなるのかを分解する・再現方法を簡潔に示す投稿が支持されている
- 冒頭で読者の悩みを的確に突く「お悩みフック」構造が反応を生みやすい。他ジャンルの例でも最初にお客様の悩みを提示することが伸びる動画の大前提になりますとされ、筋トレ領域でも「継続できない」「時間がない」等の悩み提示が転用可能
- 問いかけ・会話を誘発する投稿がコメント増加に直結する構造として定着しつつあり、問いかけ投稿でコメント増やすという切り口がThreads攻略のキーワードとして繰り返し挙げられている
- 具体的な数字・データを添えて説得力を出す型が有効。「インプレッションが1.8倍になった」なら強いという考え方が示す通り、感覚論より定量的な変化提示が刺さる
- フォロワーが少なくても伸びるチャンスがあるとされ、最初のうちは数字よりも、「誰に何を伝えたいか」を意識にして投稿してみましょうという基本姿勢が、認知拡大フェーズの方針と合致する

**自アカウント実測の所見**: 唯一の投稿（週2×30分・半年継続の話）はviews1,322に対してlikesが1と非常に低く、共感喚起の切り口自体は妥当だがCTA・問いかけがなく反応の受け皿が弱かった可能性が高い。サンプルが1件のみのため傾向断定はできず、今後は複数投稿でA/Bを取り再現性を検証する必要がある。

**改善アクション**
- [ ] 文末に「あなたはどっち派？」「何分から始めた？」等の明示的な問いかけを必ず入れ、リプライを誘発する設計に変える
- [ ] 冒頭1〜2行を「悩みフック」（例：続かない・時間がない・３日坊主）で始め、その後に価値提供を続ける構成に統一する
- [ ] 箇条書き＋絵文字で「保存したくなる」チェックリスト型投稿を週内に最低1本入れ、保存数の変化を計測する
- [ ] 感覚的な表現だけでなく「週2回」「30分」など具体的な数字を軸にした投稿を継続し、再現性のある型として固定化する

**新テンプレ草案**: 3本（data/proposed_templates.json に出力）

## 節約

**市場トレンド（Web検索による）**
- Threadsは「いいね」より会話量が重視され、共感を呼ぶ短文投稿・自己開示が伸びやすい傾向が強まっている
- リプライへの再返信（会話のラリー）を意図的に作る投稿設計がバズの起点になっている
- 冒頭2行で続きを読ませる「結果先出し型」の書き出しが節約・ライフハック系でも定番化している
- 画像より言葉（テキストのみ）の投稿の方が伸びやすく、箇条書き・絵文字での視認性向上が有効
- トピックタグを1つ設定し、話題性のあるテーマに即座に反応する「即応型」投稿がリーチを稼ぎやすい

**自アカウント実測の所見**: 自アカウントの実測データはサンプル数0件のため、伸びた投稿・伸びなかった投稿の比較分析は現時点で不可能。まずは市場トレンドに基づく仮説ベースで複数パターンを投稿し、初期データを蓄積することが優先課題。

**改善アクション**
- [ ] 会話が生まれやすい「共感の観察＋質問」型を軸に、最低週7回（1日1回）の投稿頻度を維持してアルゴリズム評価を安定させる
- [ ] 保存されやすい「箇条書き＋結果先出し」構成のテンプレを試験投稿し、保存数・リプライ数をKPIとして記録開始する
- [ ] リプライがついた投稿には運用者側からも再返信し、会話のラリーを意図的に継続させる
- [ ] リンク誘導は全体の最大1/3程度に抑え、認知拡大フェーズでは価値提供・質問系の比率を高める

**新テンプレ草案**: 3本（data/proposed_templates.json に出力）
